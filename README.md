# CustomNN.jl — Нейросетевой фреймворк на Julia

**CustomNN** — это модульный фреймворк для создания, обучения и визуализации полносвязных нейронных сетей, разработанный на языке Julia. Архитектура проекта вдохновлена современными решениями (PyTorch, Flux), обеспечивая гибкость настройки каждого этапа обучения.

---

### Подробное описание технических возможностей

Фреймворк реализован на языке **Julia**, что позволило использовать механизмы множественной диспетчеризации (Multiple Dispatch) для создания чистого и производительного API, напоминающего PyTorch, но работающего на нативных структурах данных.

#### 1. Гибкая архитектура слоев
*   **Абстракция AbstractLayer**: Все компоненты сети (полносвязные слои и функции активации) наследуются от единого интерфейса, что позволяет легко добавлять новые типы слоев.
*   **Слой Dense**: Реализует матричное умножение с эффективным кэшированием входных данных для ускорения обратного прохода (backward pass).
*   **Контейнер Sequential**: Позволяет собирать архитектуры любой глубины, автоматически управляя передачей градиентов между слоями в правильном порядке.

#### 2. Математическая база и инициализация
*   **Инициализация весов**: Реализованы алгоритмы **Glorot (Xavier) Uniform** и **He Normal**. Это критически важно для предотвращения затухания или взрыва градиентов в глубоких сетях.
*   **Численная стабильность**: В слое `Softmax` и функции `CrossEntropyLoss` используются механизмы защиты от переполнения (вычитание максимума) и логарифмирования нуля (добавление `eps`), что делает обучение устойчивым.

#### 3. Продвинутые оптимизаторы
*   **Adam (Adaptive Moment Estimation)**: Полноценная реализация с адаптивным шагом обучения для каждого параметра, поддержкой моментов (beta1, beta2) и коррекцией смещения (bias correction). Использует ленивую инициализацию состояний для экономии памяти.
*   **SGD**: Классический стохастический градиентный спуск для простых задач.
*   **Gradient Clipping**: Глобальное ограничение нормы градиента. Эта функция позволяет обучать нейросети даже на зашумленных данных, предотвращая резкие скачки весов.

#### 4. Конвейер обработки данных (Data Pipeline)
*   **DataLoader**: Итератор, который берет на себя всю логику по нарезке данных на батчи. Поддерживает автоматическое перемешивание (shuffle) перед каждой эпохой.
*   **Утилиты преобразования**: 
    *   `onehot`: Быстрое преобразование меток классов в разреженные матрицы.
    *   `train_test_split`: Инструмент для корректного разделения выборки на обучающую и проверочную с сохранением структуры данных.

#### 5. Инструменты отладки и верификации
*   **Модуль grad_check.jl**: Уникальная особенность нашего фреймворка. Позволяет проверить корректность аналитических градиентов, сравнивая их с численными производными (метод конечных разностей). Это гарантирует, что обратное распространение ошибки во всех слоях реализовано математически верно.

#### 6. Обучение и валидация
*   **Универсальный метод fit!**: Высокоуровневая функция обучения, которая берет на себя весь цикл: обнуление градиентов, прямой проход, расчет потерь, обратный проход и шаг оптимизатора.
*   **Мониторинг**: Поддержка валидационной выборки прямо в процессе обучения с выводом метрик в реальном времени.

#### 7. Визуализация и анализ
*   **Библиотека визуализации**: Интегрированные функции для построения графиков обучения (Loss/Accuracy).
*   **Анализ решающих границ**: Специальные функции для отрисовки `Decision Boundary` на плоскости (для 2D задач).
*   **Анализ важности признаков**: Метод оценки влияния входных параметров на результат на основе средних абсолютных весов первого слоя (использовано в примере с пожарами).

### Быстрый старт

**Установка зависимостей:**
Для работы требуются библиотеки Plots, XLSX, LinearAlgebra и Statistics. Запустите:
`julia install_deps.jl`

**Пример создания модели:**
```julia
model = Sequential(
    Dense(784, 256),
    ReLU(),
    Dense(256, 10),
    Softmax()
)
optimizer = Adam(lr=0.001)
loss_fn = CrossEntropyLoss()
```
